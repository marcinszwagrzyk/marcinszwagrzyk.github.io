# Refinery flaring from satellite — FIRMS active fire + Black Marble night lights

`refinery_flares.ipynb` is **GPKG-driven**: it reads refineries from
`REFINERIES_GPKG` (a GeoPackage of points or polygons with a `name` column; auto-created
from a built-in table on first run as `data/refineries_2026.gpkg`). For each site, from
2012 → today, it produces:

1. **NASA FIRMS** active-fire detections (VIIRS 375 m; MODIS 1 km optional) — the
   **primary** signal, orange bars. *Always runs.*
2. **NASA Black Marble** night-lights radiance (VNP46A4 yearly / A3 monthly) — a separate
   purple line. **Optional**: no token / unauthorized / any failure → the notebook prints a
   note and **proceeds with FIRMS only**.
3. **Satellite maps** — each refinery's FIRMS detections (coloured by year) over Esri World
   Imagery → `images/maps/`.

Time-series subplots mark the **construction start** and **production start** dates, so you
can see the signal switch on around commissioning. Outputs land in `images/` (graphs +
maps) and `data/` (`firms_monthly_all.csv`, `blackmarble_all.csv`, `timeseries_long_all.csv`,
`firms_detections.gpkg`).

## Black Marble night lights (the second series)
Black Marble (`VNP46*`) is NASA's calibrated VIIRS night-lights suite (LAADS DAAC,
collection 5200). The notebook defaults to the **yearly `VNP46A4`** product for a light
full-history trend; set `BM_PRODUCT="VNP46A3"` for monthly (heavier). To enable it:

1. **Token:** register/log in at <https://urs.earthdata.nasa.gov/> → **Profile → Generate
   Token** → paste into `BLACKMARBLE_TOKEN` in the CONFIG cell (~60-day validity).
2. **Authorize LAADS once (required):** open <https://ladsweb.modaps.eosdis.nasa.gov/> →
   **Login** with Earthdata → approve the "LAADS Web" app (and accept the data licence).
   Without this, downloads silently return an HTML login page (HTTP 303 to URS) — the
   notebook detects this and tells you to authorize.
3. `pip install h5py` (already in `requirements.txt`).

> We deliberately **do not** use the `blackmarblepy` package: it calls LAADS's retired
> `/api/v1/files` API (HTTP 404). The notebook instead lists the archive
> (`/archive/allData/5200/{product}/{year}/{doy}.json`), downloads the tile `h{H}v{V}`
> covering each site with the bearer token, and reads the BRDF-corrected NTL layer with
> `h5py` — taking the mean radiance in a `BUFFER_M`-sized window per site per timestep.
> Tiles cache under `cache/blackmarble/`. Without a token the step is skipped (FIRMS only).
> Notebook **section 5** has the full details and alternative routes (`earthaccess`, AppEEARS).

## Run
Open `refinery_flares.ipynb` and run top to bottom. The first FIRMS pull (2012→now × 7
sites) is slow; every 5-day chunk is cached under `cache/`, so re-runs are instant.
Outputs: `refinery_timeseries.png`, `refinery_fire_detections.gpkg`,
`refinery_fire_monthly.csv`, `refinery_blackmarble_monthly.csv`, `refinery_summary.csv`.

The notebook is generated by `build_notebook.py` (edit there, then re-run it to rebuild
the `.ipynb`).

## Why FIRMS for VIIRS active fire
VIIRS active fire is **not** in the Planetary Computer STAC catalog, and VIIRS only starts
in 2012. NASA FIRMS is the authoritative archive for point active-fire detections and is
ideal for attributing fires to a refinery polygon.
